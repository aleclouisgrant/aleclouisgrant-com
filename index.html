Show Control

C# | Windows Presentation Foundation
Developed for Quatrefoil Associates

The Quatrefoil Exhibit Show Control system was developed as a custom comprehensive remote control software to handle a variety of automated tasks for museum exhibits. Deployed in all of the firm’s ongoing museum projects, the system was built on the MQTT communication protocol and is routed by a Node-RED server running on a Raspberry Pi. Windows devices are connected to the MQTT server via named pipes wrapped in a Windows Service while most other devices use UDP to communicate to the Raspberry Pi. The system is primarily controlled from mobile devices using a simple UI built in Node-RED. The system was built to be easily expandable and robust as every museum's specific exhibits require unique solutions to run properly.

Features of the system include:

  • Turn on devices using Wake On Lan
  • Test connections to all devices
  • Restart applications
  • Restart devices
  • Shut down devices
  • Create a custom schedule to turn on and off all devices throughout the week
  • View logs of errors
  • View logs of usage: what the patron did on the computer, etc

The system is compatible with:

  • Windows devices
  • BrightSigns
  • Projectors
  • Smart TVs/displays
  • Smart devices
  • Anything that can communicate over UDP

Picket Fence Ranking System

C# | Windows Presentation Foundation

I designed and developed the Picket Fence Ranking System to rank professional West Coast Swing dance competitors. I created and refined a custom ELO-based algorithm that factors in the disparate levels of dancers and their partners against the competition. This creates a more fair, skill based ranking system as opposed to the current official point accumulation system.

Competitor records are stored in a Mongo database and served by a Blazor built website via web APIs. An MVVM UI application was also built in WPF for admins to handle uploading and editing data on competitors and competitions. All search boxes cross check the official World Swing Dance Council database for accurate information on competitors.

FlexCell to Google Sheets Export Library

C# | VBA | OAuth 2.0
Developed for Project Worldwide (project.com)

The FlexCell to Google Sheets Export Library is a custom library that takes FlexCell graphs and uploads the content and styling to a Google Sheet. This library was developed as part of an effort to modernize the client's legacy in house applications, so it had to be ready to drop in to their Windows Forms applications written in VBA.

This library preserves all compatible styling options including:

  • Cell background and font color
  • Grid size
  • Individual column and row sizes in width and height
  • Text alignment per cell
  • Text wrapping and overflow properties
  • Cell border colors and styles
  • And more

The library would compose a request to the Google Drive API to create the sheet then use OAuth2.0 to set the accessibility and user permissions to the appropriate Google accounts. The application could also control guest privileges and set the document to Read Only for use with the client's 3rd parties who needed access to shipping orders created via the library.

Pathfinder

C# | Windows Presentation Foundation

The Pathfinder application is built using C# and Windows Presentation Foundation in Visual Studio with an abundance of custom XAML objects to make creating grids with varied terrain quickly and dynamically. Grid size, start and goal locations, density of nontraversible spaces, and more are options available during runtime. Custom maps can be made by clicking on spaces to toggle whether they are traversible or not.

Pathfinding is implemented using the A* algorithm with 3 different distance calculation heuristics available (Manhatten, Chebyshev, or Euclidean). After the path has been calculated, each step is printed on the bottom of the screen, as well as the total steps to complete the path, and the whether the goal space is accessible from teh start space at all.

Shift Report Tool

C# | WinForms | Visual Basic | Powershell | SQL | Microsoft Access
Developed for Rutgers University, Office of Information Technology

While working at Rutgers University Office of Information Technology as a lab supervisor, I was tasked with creating a better tool to monitor employee tasks while on shift. Our old system was to have consultants send in a Google Form every 15 minutes with the task they completed since their last round. This created thousands of responses for supervisors to sort through every week which resulted in important information being missed. I pitched for the development of a standalone application more in tune with the specific needs of the labs.

Using C# and WinForms in Visual Studio, I developed a client side application that allows consultants to submit their tasks and then compiles them into a timestamped report available for consultants to review throughout the duration of their shift and send once their shift is completed.

Consultants greatly appreciated the ability to review the entirety of their rounds during their shift so they could be sure that they were taking care of all of their required responsibilities.

Supervisors can access the reports submitted by consultants through a Microsoft Access database. Custom interfaces were developed to make finding specific reports easy and fast.

Supervisors were able find information much easier with this new system implemented. While they used to have thousands of rounds to sift through each week, they now were able to see full reports and filter by consultant, date range, location, and more. This dramatically reduced the amount of time supervisors spent reading through reports for important information on the upkeep of the computer labs across campus.

Additional organizational information was scraped from various company resources using Powershell and VBA scripts.

For the development, documentation, and implementation I completed on this project, as well as the other projects I headed during my time at the University, I was awarded Supervisor Achievement Award in Fall 2017. Super Mario Sunshine Bingo Client

C++ | Qt5

C++/Qt5 application that hooks into Dolphin's emulated RAM to read values for SMS Bingo. Bingo spaces are not selectable until the proper values have been read from the game's RAM. Once the application sees the correct requirements have been met to allow the player to select the corresponding bingo space, the space turns red. The player can then select the space themselves. The application can also immediately select the corresponding space automatically once the requirements have been met. No changes to Dolphin have to be made for this to work - the program does all the work hooking to Dolphin itself.

This project is currently in development and is intended to have online multiplayer as well as a spectator client that allows for a detailed look at what each player has acquired over the course of the bingo game. This application is also intended to work with the Wii's USB port so people can play over console and cross platform (console/Dolphin).
